在dify 1.8添加ollama模型时，显示模型为空的解决办法

晚上把dify 升级到1.8版本，发现连接ollama的时候，显示模型，模型不出来，显示模型那里，始终为0个模型，在“添加模型”那里无论如何配置都没办法。

排查思路

  1. 在浏览器输入ollama，查看ollama正常，模型显示正常，说明ollama没有问题
    查看docker-compose.yml，发现这个变量是由.env控制的

在/dify/docker目录，打开.env文件，看到这个参数如下：

PLUGIN_PYTHON_ENV_INIT_TIMEOUT=120

PLUGIN_MAX_EXECUTION_TIMEOUT=600

PIP_MIRROR_URL=

Dify默认配置项的数字太小，改大即可，我修改的数字为：

PLUGIN_PYTHON_ENV_INIT_TIMEOUT=1200
PLUGIN_MAX_EXECUTION_TIMEOUT=900
PIP_MIRROR_URL=https://pypi.tuna.tsinghua.edu.cn/simple
#PIP_MIRROR_URL=

并重启dify

sudo docker-compose down
sudo docker-compose up -d

再次打开dify,模型就出来了：

MBR和GUID分区表选择指南

发表于 2025-12-04

当前的引导模式分UEFI和Legacy两种，分区表的选择与引导模式相关
UEFI对应的分区就是GUID分区(gpt);Legacy,则对应的分区类型是MBR。

win10/11支持UEFI模式优先选择GPT，老电脑或小容量硬盘选MBR

选GPT(GUID)的场景（多数新电脑）

  • win10/11系统
  • 硬盘容量>2T
  • 主板支持UEFI启动
  • 需要分更多区（超过4个主分区）
  • 追求数据安全

选MBR的场景（老设备/特殊场景）

  • Win7及以下的系统
  • 硬盘容量<2TB
  • 主板只支持Legacy BIOS启动
  • 需要兼容老设备，要装多系统，其中包含Win7及以下
  • 用于工业设备和老服务器

为什么win10/11建议采用GUID？

UEFI新的主板引导项，图形界面，启动更快，UEFI只支持64位系统且磁盘分区必须为gpt模式。

选择MBR或GUID注意什么？

  1. 启动模式和分区表必须匹配
  2. 固态硬盘（SSD）优先选GPT+UEFI：能发挥快速启动速度，支持TRIM优化，延长寿命
  3. WIN7采用UEFT+GPT安装：原生Win7不支持，采用第三方PE安装后修复引导
  4. 数据备份：转换类型会清空，要备份重要文件。

常见问题解答

  • 主板支持UEFI，需要装win7等老系统，也可以安装MBR。
  • 转换分区表类型会丢数据
  • DiskGenuis能改分区类型
  • 老电脑升级win10,原来MBR可以不改，如果硬盘<2T,启动正常。若想体验快速启动，可备份数据后转换GUID（同时改主板启动模式）
  • 选错分区表类型，电脑会开不了机，GUID分区选主板UEFI启动，MBR选Legacy启动
