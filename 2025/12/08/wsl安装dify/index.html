wsl安装dify

发表于 2025-12-08 分类于 信息技术

1.下载最新的dify版本

langgenius/dify: Production-ready platform for agentic workflow development.

2.启动dify

将dify-main.zip 上传到wsl ubuntu系统的/opt目录下并解压

3.配置转发

如果win11的wsl网络模式是mirror模式，以下可跳过

win10的wsl没有开启镜像模式，配置端口转发

  • listenport是Windows上的端口，
  • listenaddress是Windows上监听此端口的IP地址（通常为0.0.0.0以监听所有IP），
  • connectport是WSL中服务的端口，
  • connectaddress是WSL中服务的IP地址

查看端口代理

宿主机给wsl配置代理,访问网络

4.启动异常处理

新版本启动如果报错

启动后如果出现 Setup failed: PermissionDenied (persistent) at write => permission denied Context: service: fs path: privkeys/872a8d90-c43e-471c-bd9f-5bed3dbc269c/private.pem Source: Permission denied (os error 13) 报错

  1. 请创建一个/data/dify/dify_data 文件夹  权限改为777
  1. 请将docker 文件夹下面的配置文件docker-compose.ymal文件 查找./volumes/app/storage替换成/data/dify/dify_data
  2. docker compose down && docker compose up -d
